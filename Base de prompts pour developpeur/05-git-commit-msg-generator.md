# GIT Commit message generator

## INSTRUCTIONS

1. Ajouter les fichiers créés et/ou modifiés
2. Rédiger un message de commit
3. Commiter les changements

### REGLES

- Le commit doit toujours suivre la convention commitizen.
- Éviter les descriptions trop verbeuses ou les détails inutiles.
- Commencer par une phrase courte à l'impératif.
- Rester concis, idéalement moins de 50 caractères.
- Expliquer le quoi et le pourquoi, pas le comment.
- Séparer le sujet et le corps par une ligne vide.
- Utiliser le corps uniquement en cas de commit important.
- Écrire une seule phrase pour la première partie, et deux ou trois phrases au maximum pour l'explication détaillée.

### EXEMPLES DE MESSAGES

- "feat(sidebar) : ajouter le bouton utilisateur de la barre latérale"
- "fix(sidebar) : corriger le bouton utilisateur de la barre latérale"
- "refactor(sidebar) : refactoriser le bouton utilisateur de la barre latérale"
- "docs(sidebar) : ajouter la documentation pour le bouton utilisateur de la barre latérale"
- "style(sidebar) : styliser le bouton utilisateur de la barre latérale"
- "test(sidebar) : ajouter un test pour le bouton utilisateur de la barre latérale"
- "chore(sidebar) : ajouter une tâche pour le bouton utilisateur de la barre latérale"
